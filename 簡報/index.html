為什麼這章「不用 AI」？

🎯 本單元是全書的「回歸基礎」章——讓你體會：不是所有系統都需要 AI。打卡是結構化的資料操作，用資料庫就很好，硬加 AI 反而多餘。

前端 Fetch API 送 JSON 給後端 Flask 路由，後端操作 SQLite（CREATE TABLE／INSERT OR REPLACE／SELECT + 日期範圍篩選），再把結果回傳更新畫面。
💡 前後端分離：不同頁面的 JavaScript 邏輯拆到獨立 JS 檔，好維護、好擴充。

兩個實戰細節

📊

CSV 編碼陷阱

繁中在 Excel 常變亂碼——關鍵在 UTF-8 的 BOM 標記與 Big5 的取捨，匯出時處理好就不亂碼。

🖼️

照片智慧配對

照片檔名用「班級+座號」，系統自動對應到學生資料，不用一張張手動綁。

✅ 學會這章，你就掌握了「Flask + 資料庫 + 檔案處理」這套最實用的地端應用骨架，日後任何小工具都能套。
